# About Me
製造業（FA・生産技術）とITの境界で、初期設計・意思決定・立ち上げを担ってきたエンジニア。

### 思考・行動スタイル
探索起点 / 決め打ち完走タイプ（短期バイナリ報酬型）

- 不確定な状況で情報を集め、構造を把握した上で方針を一つに決めるのが得意
- 方針（YES / NO）が確定した瞬間に、最も集中力とパフォーマンスが出る
- 微差の最適化や終わりの見えない改善より、
  「決める → 実装する → 畳む」までを完走させることに価値を感じる
- 学習・理解フェーズが終了した後の長期耐久は得意ではない


---

## 得意な役割
- 初期設計 / 方針決定
- 立ち上げ・切り分け・畳み
- 現場要件と設計・システムの橋渡し
- 「決めないと止まる」場面での意思決定
- 境界が曖昧な問題を、説明可能な構造に落とすこと

### 苦手な環境（避けたい前提）
- 微差の比較を延々と続けるプロセス
- 終了条件が定義されていない改善・運用
- 判断権がない状態で責任だけが重い役割

※ 能力や努力の問題ではなく、報酬構造のミスマッチと理解しています。

---

### 技術スタンス
- 「何でもできる」より、役割を明確にして機能することを重視
- 最適化・レビュー・改善は、専門性のある人と工程分離した方が強い
- 設計や構成は、あとから他人に説明できる形で残す
- 属人化より「迷わない構造」を優先
