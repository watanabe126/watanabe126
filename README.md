# About Me

## 思考・行動スタイル
**バイナリ報酬型 / 決め打ち完走タイプ**

- 不確定な状況で情報を集め、**一つに決めて前に進める**のが得意
- YES / NO が確定した瞬間に最も集中力とパフォーマンスが出る
- 微差の最適化や終わりの見えない改善より、**方針決定・完走**に価値を感じる

---

## 得意な役割
- 初期設計 / 方針決定
- 立ち上げ・切り分け・畳み
- 現場要件と設計・システムの橋渡し
- 「決めないと止まる」場面での意思決定

---

## 苦手な環境（避けたい前提）
- 微差の比較を延々と続けるプロセス
- 終了条件が定義されていない改善
- 判断権がない状態での責任だけが重い役割

※能力や努力の問題ではなく、**報酬構造のミスマッチ**と理解しています。

---

## 技術スタンス
- 「何でもできる」より **役割を明確にして機能する**ことを重視
- 最適化・レビュー・改善は専門性のある人と**工程分離**した方が強い
- 設計や構成は、**あとから他人に説明できる形**で残す

---

## Interests
- 仮想化・分離・境界設計
- FA / 制御系とITの接続点
- 「壊れない」「迷わない」構成を作ること
